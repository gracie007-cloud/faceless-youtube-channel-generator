Framework Choice: Vanilla HTML/CSS/JS While React/Tailwind offers modern state management, a dependency-free single-file architecture was chosen for maximum portability. This app runs instantly on local browsers, GitHub Pages, Netlify, or Vercel without build pipelines, ensuring zero maintenance overhead and immediate deployment. ### 2. NotebookLM Workflow Correction A critical blind spot in AI prompt routing is misunderstanding how NotebookLM operates. NotebookLM is optimized for **document querying via its chat interface**, not for executing system prompts pasted into the "sources" box. The app now outputs a structured system prompt optimized for the **chat query field**, with clear instructions to leave sources empty (or upload niche research). This prevents instruction leakage and ensures the AI treats the prompt as a directive rather than reference material. ### 3. Session Persistence & State Recovery To prevent user frustration from accidental tab closures or refreshes, all form state is synchronized to `localStorage` in real-time. Reloading the browser automatically restores the exact step, selections, and conditional fields (like the custom name input or "Not sure" recommendations). ### 4. Instant Client-Side Fallback for "Not Sure" Rather than deferring sub-niche suggestions entirely to the external AI, the app includes an embedded data dictionary (`subSuggestions`). When a user selects *Not sure (let the AI decide)*, three highly validated sub-niche options instantly appear based on their selected primary niche. This provides immediate value and improves the downstream AI context window accuracy. ### 5. Pre-Computed YouTube Metrics The prompt engine dynamically injects pre-calculated performance benchmarks into the generated blueprint request: - **Target Words per Script:** Mapped to selected runtime lengths - **Retention Goals:** Average Viewer Duration (AVD) targets per duration tier - **Estimated CPM Ranges:** Industry-standard ad revenue brackets per niche These metrics constrain the AI's output, forcing it to generate realistic, production-ready scripts and monetization timelines instead of generic advice. ### 6. Accessibility & Navigation Standards - **ARIA Compliance:** All interactive elements use `aria-pressed`, `aria-label`, and semantic roles. Screen readers accurately announce current progress and selection states. - **Keyboard Navigation:** Full `Tab`/`Enter`/`Space` support for custom radio/checkbox cards. Focus rings are explicitly styled for high contrast. - **Validation Gating:** The "Continue" button remains disabled until required fields are populated, replacing disruptive browser alerts with native UI state management. - **Section Delimiters:** The generated prompt uses explicit `SECTION 1-3` headers. This structural parsing optimization prevents context window bleeding and forces strict adherence to the requested output format. ## 🚀 Deployment Instructions 1. **Local Testing:** Save the code as `index.html` and double-click to open. 2. **Free Hosting:** - Create a GitHub repository named `faceless-youtube-blueprint` - Add `index.html` to the root - Enable GitHub Pages in repository settings - App will be live at `https://.github.io/faceless-youtube-blueprint/` 3. **Custom Domain:** Point DNS to Netlify/Vercel via drag-and-drop deployment. ## 📋 Sample End-to-End Workflow 1.
